이번 주 AI 연구 하이라이트: 에이전트는 어떻게 더 똑똑해지고 있는가

2026-04-28 · Blog Writer연구에이전트강화학습도구사용

AI 에이전트가 외부 도구를 사용하고, 스스로 계획을 세우며, 팀을 이루어 협업하는 시대가 본격화되고 있다. 이번 주 arXiv에 공개된 최신 연구 6편을 분석했다. 공통적으로 드러나는 세 가지 트렌드는 강화학습(RL) 기반 에이전트 정렬, 멀티 에이전트 시스템의 신뢰성 확보, 그리고 도구 사용의 스케일업이다.

1. 강화학습으로 에이전트를 정렬하다

R2IF: 작은 모델도 정확하게 도구를 호출할 수 있다

LLM이 외부 API를 호출할 때, "왜 이 도구를 선택했는지" 설명하는 추론(Chain-of-Thought)과 실제 호출이 일치하지 않는 문제가 있었다. R2IF는 이 간극을 세 가지 보상 신호를 결합한 RL로 해결한다. 추론이 올바른 도구 선택에 실제로 기여했는지 측정하는 CER(CoT Effectiveness Reward)이 핵심이다.

결과는 인상적이다. Llama 3.2-3B 같은 소형 모델에서 기존 대비 +34.6% 성능 향상을 달성했다. 이는 고가의 대형 모델 없이도 신뢰할 수 있는 도구 호출 에이전트를 구축할 수 있음을 시사한다.

📄 Cheng et al., "R2IF: Aligning Reasoning with Decisions via Composite Rewards for Interpretable LLM Function Calling" (2026.4.22)

Plan-RewardBench: 에이전트의 궤적을 평가하는 새로운 기준 (ACL 2026)

에이전트가 도구를 여러 번 호출하며 문제를 해결하는 전체 과정(궤적)을 평가하는 보상 모델(Reward Model)은 아직 초기 단계다. Plan-RewardBench는 이 분야 최초의 전문 벤치마크로, 안전 거부·도구 관련성·복잡 계획·오류 복구 등 4개 카테고리에서 1,171개 평가 쌍을 제공한다.

충격적인 발견은 현재 최고 성능 모델도 정확도 70% 미만이라는 점이다. 특히 32,000 토큰을 넘는 장기 궤적에서는 대부분의 모델이 랜덤 수준으로 하락한다. 에이전트를 RLHF로 학습시키려면, 먼저 궤적을 제대로 평가할 수 있는 보상 모델부터 개선해야 한다는 경고다.

📄 Wang et al., "Aligning Agents via Planning: A Benchmark for Trajectory-Level Reward Modeling" (2026.4.9, ACL 2026 승인)

2. 적대적 환경에서도 살아남는 멀티 에이전트

The Consensus Trap: 과반수가 공격당해도 정답에 도달하는 법

멀티 에이전트 시스템에서 다수결 투표는 직관적이지만 치명적인 약점이 있다. 에이전트의 절반 이상이 해킹되거나 오작동하면, 투표 결과 자체가 오염된다. The Consensus Trap은 각 에이전트가 토큰 단위로 번갈아 공유 컨텍스트에 기여하는 "라운드 로빈" 방식을 제안한다.

핵심 아이디어는 간단하다. 잘못된 논리는 중간에 차단될 수 있지만, 완성된 잘못된 답변은 되돌리기 어렵다. 5개 에이전트 중 3개가 손상된 극단적 상황에서도 다수결(0.8%)에서 라운드 로빈(89.1%)으로 88%p 향상을 달성했다. 추가 연산 비용은 없다.

📄 Liu et al., "The Consensus Trap: Rescuing Multi-Agent LLMs from Adversarial Majorities via Token-Level Collaboration" (2026.4.18)

3. 수백 개 도구를 다루는 에이전트

HTAA: 도구를 "팀"으로 묶어 관리한다

실제 기업 환경에서 에이전트는 수백 개의 API를 다뤄야 한다. 모든 도구를 한꺼번에 제시하면 LLM의 컨텍스트가 폭발하고, 잘못된 도구를 선택할 확률이 높아진다. HTAA는 자주 함께 쓰이는 도구들을 "에이전트 도구"로 묶어 계층화한다.

라이드헤일링 서비스의 POI 검증 업무에 적용한 결과, 수작업 부담 84.5% 감소, 주석 비용 81.25% 절감, 토큰 사용량 50.6% 감소를 달성했다. 대형 도구 생태계를 가진 기업이라면 주목할 만한 접근법이다.

📄 Huang et al., "HTAA: Enhancing LLM Planning via Hybrid Toolset Agentization & Adaptation" (2026.4.13)

ToolsRL: 이미지를 확대하고 회전하는 법을 스스로 배우다

멀티모달 LLM은 이미지를 "보기"만 하지, 능동적으로 조작하지는 못했다. ToolsRL은 확대(zoom-in), 회전, 포인트 표시 등 시각 도구를 RL로 학습하는 2단계 커리큘럼을 제안한다. 1단계에서 도구 조작 기술을 익히고, 2단계에서 실제 문제 해결에 활용한다.

단일 단계로 학습하면 모델이 도구를 무시하고 텍스트만으로 추론하려는 경향이 생기는데, 커리큘럼 접근이 이를 방지한다. 문서 이해에서 기존 대비 +16%, 공간 추론에서 95.6% 정확도를 달성했다.

📄 Dong et al., "Visual Reasoning through Tool-supervised Reinforcement Learning" (2026.4.21)

보너스: AI가 AI 연구를 자동화한다

AutoSOTA: 논문에서 SOTA 모델을 자동으로 재현하고 개선

Tsinghua와 Microsoft Research의 AutoSOTA는 8개 전문 에이전트가 논문 파싱부터 코드 재현, 가설 생성, 실험 모니터링까지 전 과정을 수행하는 연구 자동화 시스템이다. NeurIPS, ICLR 등 주요 학회 논문을 대상으로 105개의 새로운 SOTA 모델을 발견했으며, 논문당 평균 5시간이 소요됐다. "Red Line System"으로 cherry-picking이나 무효 메트릭 향상을 방지한 점도 주목할 만하다.

📄 Li et al., "AutoSOTA: An End-to-End Automated Research System for State-of-the-Art AI Model Discovery" (2026.4.7)

이번 주의 시사점

이번 주 연구들이 공통으로 가리키는 방향은 명확하다. 에이전트의 "행동"을 직접 최적화하는 RL이 SFT를 넘어 핵심 학습 패러다임으로 자리잡고 있으며(R2IF, ToolsRL, Plan-RewardBench), 에이전트가 실제 환경에서 안전하고 신뢰할 수 있게 동작하기 위한 연구(Consensus Trap, Plan-RewardBench의 Safety Refusal)가 빠르게 성장하고 있다. 수백 개 도구를 효율적으로 관리하고(HTAA), 연구 자체를 자동화하는(AutoSOTA) 시도는 에이전트 기술이 이미 산업 적용 단계에 진입했음을 보여준다.
