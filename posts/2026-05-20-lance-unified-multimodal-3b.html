Lance: 3B 파라미터로 이미지·영상 이해+생성+편집을 통합한 멀티모달 AI

AI에게 사진을 보여주고 "이게 뭐야?"라고 물으면 대답하는 모델과, "이런 사진 만들어줘"라고 요청하면 생성하는 모델은 지금까지 별개의 시스템이었다. 바이트댄스(ByteDance) 연구진이 이 두 능력을 하나로 합치면서도 파라미터 수를 30억 개(3B)로 억제한 모델 'Lance'를 공개했다. 70억 이상의 경쟁 모델보다 작은 규모로 이미지·영상의 이해, 생성, 편집을 동시에 수행하며 여러 벤치마크에서 기존 통합 모델을 앞선다.

연구 배경: 왜 '통합'이 어려운가

멀티모달 AI — 텍스트, 이미지, 영상 등 여러 종류의 데이터를 다루는 AI — 는 크게 두 갈래로 발전해 왔다. 하나는 이해(understanding), 이미지나 영상을 보고 질문에 답하거나 설명하는 능력이다. 다른 하나는 생성(generation), 텍스트 지시에 따라 이미지나 영상을 만들어내는 능력이다. 각각 별도의 모델 구조와 학습 데이터가 필요했기 때문에, 하나의 모델이 두 능력을 모두 갖추려면 파라미터가 70억(7B) 이상으로 커지는 것이 일반적이었다.

문제는 두 능력이 서로 충돌하기 쉽다는 점이다. 이해에 최적화된 내부 표현과 생성에 최적화된 표현은 방향이 다르다. 하나의 모델에 두 목표를 동시에 학습시키면 양쪽 모두 성능이 떨어지는 '다중 과제 간섭(multi-task interference)' 현상이 발생한다.

핵심 방법론: 이중 전문가 구조와 단계적 학습

Lance는 이 충돌을 세 가지 설계 선택으로 해결한다.

이중 스트림 전문가 혼합

하나의 트랜스포머(대규모 언어모델의 기본 구조) 위에 '이해 전문가'와 '생성 전문가'를 병렬로 배치했다. 입력 데이터가 텍스트·이미지·영상이 섞인 하나의 시퀀스로 들어오지만, 내부적으로는 의미를 파악하는 경로와 시각적 결과물을 만드는 경로가 분리된다. 각 전문가가 자신의 영역에 집중하면서도 정보를 공유할 수 있는 구조다.

모달리티 인식 위치 부호화(MaPE)

이미지 토큰, 영상 프레임, 텍스트 토큰은 각각 고유한 위치 정보 체계를 사용한다. Lance는 '모달리티 인식 회전 위치 부호화(MaPE)'를 도입하여 서로 다른 종류의 토큰이 위치 정보 차원에서 간섭하지 않도록 했다. 영상의 시간 축까지 자연스럽게 처리할 수 있다.

단계적 다중 과제 학습

모든 과제를 처음부터 동시에 학습시키는 대신, 이해 → 이미지 생성 → 영상 생성 → 편집의 순서로 단계적으로 능력을 쌓아 올린다. 각 단계마다 이전 능력이 퇴화하지 않도록 관리한다.

주요 결과

Lance는 30억 파라미터만으로 70억 이상의 경쟁 모델과 맞서거나 이를 넘어서는 성능을 보여줬다.

벤치마크측정 대상Lance(3B) 점수비교
GenEVAL이미지 생성 품질0.907B+ 통합 모델과 동등
DPG-Bench복잡 프롬프트 이행84.67관계 접지 93.38
GEdit-Bench이미지 편집7.30통합 모델 중 최고
VBench영상 생성85.11통합 모델 카테고리 1위
MVBench영상 이해62.0통합 모델 중 최고

영상 편집에서는 배경 교체, 객체 조작, 스타일 변환 등 다양한 작업을 대화형으로 수행할 수 있다. "배경을 겨울로 바꿔줘" → "인물의 옷 색상을 파란색으로" → "전체를 수채화 스타일로"처럼 순차적 편집 명령을 처리하면서 일관성을 유지한다.

의미와 전망

Lance의 핵심 기여는 효율성과 통합성의 양립이다. 기존에는 통합 모델을 만들려면 파라미터를 대폭 늘려야 했는데, Lance는 전문가 분리 구조와 단계적 학습으로 3B 규모에서도 경쟁력 있는 성능을 달성했다. 128대 이하의 GPU로 학습이 가능하다는 점은 학계와 중소 기업에 중요한 접근성 향상을 의미한다.

하나의 모델로 콘텐츠 이해·생성·편집 파이프라인을 구축할 수 있어, 영상 제작 도구나 크리에이터 플랫폼에서의 응용이 기대된다.

다만 3B 모델의 특성상 초고해상도 영상이나 매우 긴 영상 생성에서는 대형 모델 대비 한계가 있을 수 있다. 벤치마크 비교 대상이 주로 오픈소스 통합 모델이며, GPT-4o나 Gemini 같은 폐쇄형 모델과의 직접 비교는 제한적이다. 연구진이 모델 가중치와 코드를 공개하여 후속 연구와 검증이 가능하다.
