멀티모달 통합부터 추론 혁신까지: 이번 주 주목할 AI 논문 5편

2026-04-28 · Blog Writer연구오픈소스산업동향

지난 한 주(4/21~4/24) 동안 공개된 AI 연구 논문 중, 멀티모달 AI, 에이전트 시스템, 추론 효율성 분야에서 주목할 만한 5편을 선별하여 분석했다. 이번 주의 키워드는 **"통합과 효율"**이다. 서로 다른 모달리티를 하나의 모델로 합치려는 시도, 에이전트의 도구 사용을 최적화하려는 노력, 그리고 추론 과정 자체를 압축하려는 연구가 동시에 진행되고 있다.

1. 멀티모달 통합의 두 갈래: 처음부터 하나 vs. 기존 모델 연결

멀티모달 AI 분야에서 이번 주 가장 흥미로운 대비를 보여준 두 논문이 있다. 하나는 처음부터 이해와 생성을 통합하는 새로운 아키텍처를 제안하고, 다른 하나는 이미 잘 만들어진 기존 모델들을 효율적으로 연결하는 실용적 접근을 택했다.

LLaDA2.0-Uni: 확산 모델로 이해와 생성을 동시에

대부분의 대규모 언어 모델(LLM)은 토큰을 순차적으로 생성하는 autoregressive 방식을 사용한다. LLaDA2.0-Uni는 이 패러다임에서 벗어나, 마스크 확산(masked diffusion) 메커니즘을 텍스트와 이미지 양쪽에 적용한다. 16B 파라미터 규모의 MoE(Mixture of Experts) 아키텍처 위에, SigLIP-VQ 토크나이저로 이미지를 이산 토큰으로 변환하고, 확산 디코더로 다시 고품질 이미지를 복원하는 구조다.

결과적으로 이미지 이해, 생성, 편집을 단일 모델에서 수행할 수 있게 되었다. 이해 능력은 전문 VLM(Vision-Language Model) 수준에 근접하고, 인터리브드 대화 중 이미지를 생성하거나 편집하는 것도 가능하다. 코드와 모델 가중치가 오픈소스로 공개되어 재현이 가능하다는 점도 주목할 만하다.

다만 16B 규모이므로 100B+ 모델 대비 성능 격차가 있을 수 있고, 이미지 생성 품질은 최신 독점 모델에는 미치지 못한다. 피어리뷰 이전의 기술 보고서라는 점도 고려해야 한다.

MMCORE: 기존 모델을 30% 비용으로 연결하는 실용적 경로

LLaDA2.0-Uni가 처음부터 통합 모델을 구축했다면, MMCORE는 이미 훈련된 VLM과 확산 모델(diffusion model)을 깊은 융합 없이 연결하는 방법을 제안한다. 핵심은 학습 가능한 쿼리 토큰을 통해 VLM의 시맨틱 임베딩을 예측하고, 이를 확산 모델의 조건(conditioning)으로 활용하는 것이다.

이 접근의 최대 장점은 효율성이다. 통합 모델을 처음부터 학습하는 것 대비 약 30%의 계산 비용으로 유사한 결과를 얻는다. DreamBench 벤치마크에서 GPT-4o 정렬 점수 0.8585, 텍스트-이미지 생성과 편집 양쪽에서 기존 베이스라인 대비 유의미한 개선을 보였다.

두 논문의 대비는 멀티모달 통합 트렌드의 핵심 긴장을 보여준다. 순수한 아키텍처 혁신과 실용적 효율성 사이에서 산업계가 어떤 선택을 할지가 향후 제품 형태를 결정할 것이다.

2. 에이전트의 도구 사용: 학습하는 법과 확장하는 법

AI 에이전트가 외부 도구를 얼마나 잘 다루느냐는 실제 배포 가능성을 좌우하는 핵심 요소다. 이번 주에는 도구 사용 능력을 학습하는 연구와 도구 사용 비용을 줄이는 연구가 나란히 등장했다.

ToolsRL: 강화학습으로 시각 도구 다루기 (CVPR 2026)

ToolsRL은 멀티모달 언어 모델에게 시각 도구(확대, 회전, 뒤집기, 그리기 등) 사용법을 강화학습으로 가르치는 프레임워크다. 핵심 아이디어는 2단계 커리큘럼 학습이다. 첫 번째 단계에서 도구 자체의 역학을 집중 학습하고, 두 번째 단계에서 최종 태스크 정확도를 최적화한다. 이렇게 분리함으로써 이종 태스크 간 최적화 충돌을 방지한다.

결과는 압도적이다. DocVQA에서 이전 SOTA 대비 +16 포인트(61.3%→77.3%), V-Star에서 +4.3포인트(88.2%→92.5%), ChartQA-Pro에서 +4.6포인트를 기록했다. 샘플당 평균 3.4회의 도구 호출로, 경쟁 모델보다 훨씬 적극적인 다단계 도구 활용 패턴을 보인다. CVPR 2026 승인 논문이라는 점에서 결과의 신뢰도가 높다.

이 연구의 커리큘럼 학습 방식은 시각 도구를 넘어, 코드 실행이나 API 호출 등 다양한 도구 세트로 일반화될 가능성이 있다. 다만 도구별 보상 함수를 수동으로 설계해야 한다는 점은 확장의 병목이 될 수 있다.

Tool Attention: MCP "도구 세금"을 95% 줄이다

MCP(Model Context Protocol)가 에이전트 도구 표준으로 자리잡으면서 새로운 문제가 부상했다. 도구가 많아질수록 매 턴마다 전체 도구 스키마를 주입해야 해서 10,000~60,000 토큰이 낭비된다. 이른바 "Tools Tax" 문제다.

Tool Attention은 이를 3단계로 해결한다. (1) 사용자 의도와 도구 설명의 시맨틱 유사도를 계산하여 후보를 선별하고, (2) 사전조건(인증, 접근 범위)을 확인하는 게이팅 함수로 부적격 도구를 제외하며, (3) 통과한 도구만 전체 스키마를 온디맨드로 로드한다.

120개 도구 시뮬레이션에서 턴당 토큰이 47,312에서 2,368로 95% 감소했고, 태스크 성공률은 72%에서 94%로 오히려 상승했다. 태스크당 비용은 $0.21에서 $0.03로 86% 절감이 예상된다.

주의할 점: 성공률과 비용 수치는 토큰 절감량에서 투사(projection)한 추정치이지 실제 라이브 에이전트 환경에서의 측정치가 아니다. 피어리뷰 전 프리프린트라는 점과 함께, 실환경 검증이 필요하다. 그러나 문제 정의와 접근법 자체는 MCP를 활용하는 모든 에이전트 시스템에 직접적으로 관련된다.

3. 자연어 없이 생각하기: 추론 효율의 패러다임 전환

Abstract-CoT: 추상 토큰으로 추론 비용 11.6배 절감

Chain-of-Thought(CoT) 추론은 LLM 성능을 크게 끌어올렸지만, 긴 자연어 추론 과정은 비용과 지연의 주범이다. 이 논문은 근본적인 질문을 던진다: 추론이 반드시 자연어로 이루어져야 하는가?

Abstract-CoT는 자연어 대신 예약된 어휘의 이산 추상 토큰 시퀀스를 생성하여 추론한다. 기존 자연어 CoT를 마스킹하고 추상 토큰으로 압축하는 self-distillation 과정을 거치면, 모델이 자체적인 "추론 언어"를 학습하게 된다. 흥미롭게도, 이 추상 어휘에서 자연어와 유사한 power law 분포가 자발적으로 나타난다.

수학 추론, 지시 수행, 멀티홉 추론에서 자연어 CoT와 동등한 성능을 유지하면서 추론 토큰을 최대 11.6배 절감한다. o1/o3 계열처럼 긴 CoT를 생성하는 모델에서 이 기술의 가치가 극대화된다.

해석 가능성(interpretability) 저하는 피할 수 없는 트레이드오프다. "모델이 무슨 생각을 하는지" 자연어로 확인할 수 없게 되면 디버깅과 안전성 감사가 어려워진다. 그러나 추론 비용이 LLM 운영의 핵심 비용인 현실에서, 11.6배 절감은 무시하기 어려운 수치다.
