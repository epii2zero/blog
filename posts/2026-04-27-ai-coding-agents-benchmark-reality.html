AI 코딩 에이전트의 진짜 실력: 벤치마크 너머의 현실

Google이 "신규 코드의 75%가 AI 생성"이라 밝히고, Cursor가 500억 달러 밸류에이션을 기록하는 시대다. AI 코딩 도구는 이미 개발자의 일상이 됐고, 산업은 이들의 역량을 당연시하고 있다. 하지만 최신 학술 연구들은 다른 이야기를 하고 있다. 벤치마크 점수 뒤에 숨겨진 현실을 들여다보자.

AI가 개발자를 빠르게 만드는가? 데이터는 "아직"이라 말한다

METR(Model Evaluation & Threat Research)이 수행한 연구는 이 질문에 가장 엄밀한 답을 내놓았다. 경험 많은 오픈소스 개발자 16명을 대상으로 무작위 대조 시험(RCT)을 설계한 이 연구에서, AI 도구(Cursor Pro + Claude 3.5/3.7 Sonnet)를 사용한 개발자들은 오히려 19% 더 오래 걸렸다.

더 흥미로운 점은 개발자들의 자가 평가다. 그들은 20% 빨라졌다고 인지했다. 실제 속도와 체감 속도 사이의 괴리는, AI 코딩 도구에 대한 산업의 낙관론이 실증보다 감각에 기반할 수 있음을 시사한다.

물론 표본 크기(16명)가 작다는 한계가 있고, 2026년 2월 후속 연구에서는 선택 편향 문제도 인정됐다. METR은 현재 실험 설계를 재구성 중이다. 하지만 AI 도구의 급속한 발전 속에서도 "생산성 향상"을 엄밀하게 입증한 연구가 아직 없다는 사실 자체가 의미심장하다.

벤치마크의 함정: 버그 수정과 기능 개발은 다르다

AI 코딩 에이전트의 역량을 측정하는 표준 벤치마크인 SWE-bench에서, 최신 모델들은 70%를 넘는 해결률을 보여준다. 이 숫자만 보면 AI가 곧 개발자를 대체할 것 같지만, 두 편의 새로운 연구가 이 환상을 깬다.

FeatureBench(ICLR 2026)는 단일 버그 수정이 아닌 end-to-end 기능 개발을 평가한다. 여러 파일을 수정하고, 새 모듈을 만들고, API를 설계해야 하는 실제 개발 업무에 가까운 태스크다. 결과는 충격적이다. SWE-bench에서 74.4%를 달성한 Claude 4.5 Opus가 FeatureBench에서는 11.0%에 그쳤다.

비슷한 이야기가 SWE-EVO에서도 반복된다. 이 벤치마크는 릴리즈 노트 기반의 장기 소프트웨어 진화 시나리오를 평가하는데, 평균 21개 파일에 걸친 다단계 수정을 요구한다. SWE-bench에서 72.8%를 달성한 GPT-5.4 + OpenHands 조합이 SWE-EVO에서는 25%만 해결했다.

이 두 벤치마크가 보여주는 패턴은 명확하다. 현재 AI 코딩 에이전트는 잘 정의된 단일 이슈를 수정하는 데는 뛰어나지만, 실제 소프트웨어 개발의 복잡성 — 다중 파일 수정, 아키텍처 결정, 장기적 코드 진화 — 앞에서는 아직 갈 길이 멀다.

보안: 평가 방법조차 믿을 수 없다

AI가 생성하는 코드의 양이 폭발적으로 늘어나면서, 보안 문제도 새로운 국면을 맞이하고 있다. 그런데 최신 연구들은 보안 문제 자체보다 더 근본적인 문제를 제기한다. 보안을 평가하는 방법 자체가 불완전하다는 것이다.

ICSE 2026에서 발표된 연구("Rethinking the Evaluation of Secure Code Generation")는 기존 보안 코드 생성 기법들의 평가에 체계적 결함이 있음을 밝혔다. 핵심 문제는 보안성과 기능 정확성을 분리된 데이터셋으로 따로 평가하는 관행이다. 보안 전용 데이터에서 보안만, 일반 데이터에서 기능만 평가하면 트레이드오프가 숨겨진다. 통합 평가를 적용하자 기존 기법들의 보고된 개선 효과가 크게 축소됐다.

더불어 업계 표준 정적 분석기인 CodeQL이 다수의 취약점을 탐지하지 못한다는 사실도 밝혀졌다. 단일 도구에 의존한 보안 검증의 위험성을 경고하는 대목이다.

한편, SecRepoBench는 실제 C/C++ 리포지토리의 보안 취약점 수정 이력에서 추출한 318개 태스크로 구성된 벤치마크다. 긍정적인 발견도 있었다. 리포지토리 컨텍스트를 탐색하고 도구를 활용할 수 있는 코드 에이전트가 독립형 LLM보다 보안 측면에서 우수한 성능을 보인 것이다. 이는 Cursor나 Devin 같은 에이전트 기반 아키텍처가 보안 측면에서도 유효한 방향임을 시사한다.

낙관과 현실 사이

AI 코딩 도구를 부정할 이유는 없다. 코드 자동완성, 단위 테스트 생성, 간단한 버그 수정 같은 영역에서 이미 실질적 가치를 제공하고 있다. 하지만 최신 연구들이 일관되게 보여주는 메시지는 분명하다.

  • 벤치마크 점수 ≠ 실무 역량: SWE-bench 70%+ 해결률은 인상적이지만, 실제 개발 업무의 복잡성을 반영하지 않는다
  • 생산성 향상은 미입증: 체감과 실측의 괴리는 산업 전체의 AI 도구 ROI 평가를 재검토하게 만든다
  • 보안 평가 인프라가 미성숙: AI 생성 코드가 급증하는 상황에서, 보안 검증 방법론의 신뢰성 확보가 시급하다

AI 코딩 에이전트는 빠르게 발전하고 있고, 오늘의 한계가 내일의 한계는 아닐 것이다. 하지만 벤치마크 점수와 기업 마케팅의 낙관 뒤에 어떤 현실이 있는지 아는 것은, 이 도구들을 더 현명하게 사용하는 첫걸음이다.
